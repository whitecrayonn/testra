## 4. Decisions Made

- **Non-JS apps get package.json wrappers:** Go (api, worker) and Python (ml) apps now have `package.json` files with `dev`/`build`/`test`/`lint` scripts that invoke the appropriate toolchain via cross-platform Node.js wrapper scripts. This allows Turborepo to orchestrate all apps uniformly.
- **Cross-platform Node.js scripts instead of shell scripts:** All dev orchestration scripts are `.mjs` files using Node.js APIs, replacing the previous PowerShell-only approach. This ensures macOS/Linux/Windows compatibility.
- **`pnpm dev` starts everything:** The root `dev` script runs `start-infra.mjs` (checks local services + migrations) then `turbo run dev` (all apps simultaneously). No Docker or manual commands needed.
- **`postinstall` auto-sets up Python:** `pnpm install` automatically creates a `.venv` in `apps/ml` and installs ML dependencies. Gracefully skips if Python 3.12+ is not found.
- **Air auto-detection for Go hot reload:** The `go-dev.mjs` script checks if `air` is installed and uses it if available; otherwise falls back to `go run`. No hard dependency on air.
- **No changes to product architecture or implementation logic:** All changes are confined to dev tooling, package.json wrappers, and documentation.

## 8. Risks

- **Python not installed on dev machine:** `setup-python.mjs` gracefully skips ML venv creation. ML service will not start with `pnpm dev` until Python 3.12+ is installed. This is documented in README prerequisites.
- **Air not installed:** Go apps fall back to `go run` (no hot reload). Documented as optional in README.
- **Local services not running:** `start-infra.mjs` will fail with a clear error message listing required services. Docker Compose is available as an optional fallback.
- **Cross-platform compatibility:** Dev scripts use Node.js APIs for Windows/macOS/Linux compatibility.
- **Turbo schema warning:** IDE may show "Unable to load schema from https://turbo.build/schema.json" — this is an IDE trust setting, not a functional issue.

## 10. Important Notes

- **No product architecture or implementation logic was changed.** All modifications are developer-experience tooling, package.json wrappers, and documentation.
- **Python venv auto-setup is graceful:** If Python is not found, `pnpm install` still succeeds — the ML service simply won't start until Python 3.12+ is installed.
- **Go hot reload is opt-in:** Install `air` via `go install github.com/air-verse/air@latest` for hot reload. Without air, Go apps use `go run` (manual restart on change).
- **`pnpm dev` flow:** (1) start-infra.mjs checks local services (PostgreSQL, Redis), (2) runs migrations, (3) hands off to `turbo run dev` which launches all 4 apps + 4 shared packages simultaneously. No Docker required (ADR-009).
- **Service ports:** API 8080, Web 3000, ML 8000, PostgreSQL 5432, Redis 6379, ClickHouse 8123/9000, MinIO 9002/9001, Mailpit 1025/8025.
- **Legacy scripts preserved:** `scripts/dev/up.ps1`, `down.ps1` remain as optional Docker Compose helpers. `migrate.ps1`, `build-api.ps1` remain for backward compatibility.
